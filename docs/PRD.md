# PRD: 체육관 체크인 시스템

## 목표
체육관 회원이 입구 공용 태블릿에서 스스로 본인을 찾아 체크인하게 하여, 코치가 이름을 묻고 수동 출석 체크하는 반복 작업을 제거한다. 여러 지점을 하나의 시스템에서 중앙 관리한다.

## 사용자
- **회원** — 체육관 이용자. 입구 공용 태블릿에서 음성 또는 타이핑으로 본인 확인 후 체크인. 앱 설치·로그인 없음.
- **관리자(사장님/코치)** — 반응형 웹앱에서 회원·회원권·출석 이력 관리. PC와 모바일 둘 다 지원.
  - **전역 관리자**: 전 지점 조회·관리, 지점 생성/수정/삭제, 전 회원 대상 대량 만료 연장.
  - **지점 관리자**: 자기 지점 데이터만 조회·관리.

## 핵심 기능
1. **태블릿 지점 초기화** — 관리자 로그인 후 지점 선택 → `localStorage`에 저장 → 이후 회원 체크인 화면 기본 진입.
2. **회원 체크인** — 입력 방식 선택(음성/타이핑). 음성은 Web Speech API(한국어), 3회 연속 실패 시 타이핑으로 자동 전환. 타이핑은 이름 텍스트 + 전화번호 숫자패드. 검색 결과에서 본인 선택 → 회원권 상태 확인 → 체크인 완료(같은 날 중복 허용, 체크인은 자기 소속 지점에서만).
3. **관리자 회원 CRUD** — 필드: 이름, 전화번호, 생년월일, 소속 지점.
4. **관리자 회원권 관리** — 부여(월정액 / 횟수권 10회·기한 2달), 정지(시작일·종료일·사유 입력 → 만료일이 기간만큼 연장), 환불(사유 필수, 즉시 만료 처리). 변경 기능은 없음.
5. **대량 만료 연장** — 전역 관리자 전용. 지점·회원권 유형 필터 + 연장 일수 + 사유로 매칭된 활성 회원권 end_date를 일괄 연장(예: 연휴 3일 일괄 추가). 승인 절차·알림 없음.
6. **체크인 이력** — 오늘/기간별 조회. 1일 1회 집계 옵션 제공.
7. **관리자 최초 비밀번호 변경 강제** — `must_change_password=true`인 계정은 로그인 직후 비번 변경 화면 외 접근 차단.
8. **지점 관리** — 전역 관리자가 지점 CRUD.

## MVP 제외 사항
- 회원이 개인 스마트폰으로 체크인
- 수업/프로그램·코치 지정
- 결제 / 구독 자동화
- SMS·카카오 알림
- 지점 간 크로스 체크인
- 오프라인 큐잉(네트워크 단절 시 체크인)
- 통계 대시보드
- 네이티브 앱(관리자 UI는 반응형 웹앱으로 대체)

## 디자인
- **키오스크(회원 화면)**: 큰 터치 타겟, 1초 만에 이해되는 버튼, 짧은 체류 시간. 고대비.
- **관리자(웹앱)**: 도구처럼. 데이터 테이블·폼 중심. 반응형으로 PC·모바일 모두 지원.
- **색상**: 무채색 배경 + 포인트 1색. 보라/인디고·그라디언트 텍스트·glass morphism 금지.
