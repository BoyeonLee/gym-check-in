# PRD: 체육관 체크인 시스템

## 목표
체육관 회원이 입구 공용 태블릿에서 스스로 본인을 찾아 체크인하게 하여, 코치가 이름을 묻고 수동 출석 체크하는 반복 작업을 제거한다. 여러 지점을 하나의 시스템에서 중앙 관리한다.

## 사용자
- **회원** — 체육관 이용자. 입구 공용 태블릿에서 음성 또는 타이핑으로 본인 확인 후 체크인. 앱 설치·로그인 없음.
- **관리자(사장님/코치)** — 반응형 웹앱에서 회원·회원권·출석 이력 관리. PC와 모바일 둘 다 지원.
  - **전역 관리자**: 전 지점 조회·관리, 지점 생성/수정/삭제, 전 회원 대상 대량 만료 연장.
  - **지점 관리자**: 자기 지점 데이터만 조회·관리.

## 핵심 기능
1. **태블릿 지점 초기화** — 관리자 로그인 후 지점 선택 → `localStorage`에 저장 → 이후 회원 체크인 화면 기본 진입.
2. **회원 체크인** — 입력 방식 선택(음성/타이핑). 음성은 Web Speech API(한국어), 3회 연속 실패 시 또는 마이크 권한 거부·브라우저 미지원 시 즉시 타이핑으로 자동 전환. 타이핑은 **이름 텍스트(prefix 일치, 최소 2글자)** / **전화번호 뒷자리 4자리(숫자패드, 정확 일치)** / **회원 번호(숫자패드, 정확 일치)** 세 탭 중 회원이 선택. 검색 결과는 **활성 회원권(`status='active'`)이 있는 회원만** 노출하며, 그 외(없음/expired/refunded/paused)는 검색 자체에서 제외된다. 검색 결과에서 본인 선택(동명이인이 2명 이상일 때만 선택 화면, **최근 체크인 순으로 정렬**, 식별 보조 정보는 마스킹 — 회원 번호 `#1234`, 전화 `010-****-1234`, 생년월일 `**-04-15`) → 체크인 완료(같은 날 중복 가능, 체크인은 자기 소속 지점에서만). **횟수권 잔여 차감은 동일 회원·동일 날짜의 첫 체크인일 때만 1회**(같은 날 두 번째부터는 기록만 남고 잔여 변동 없음). 잔여가 0이 되는 순간 같은 트랜잭션에서 `status='expired'`로 즉시 전환. 체크인 화면에는 **오늘 해당 지점 체크인 인원 수**를 함께 표시. **`InputSelect/VoiceSearch/TypingSearch/MemberPick`에서 10초 무입력이면 Idle(대기 화면)으로 자동 복귀.**
3. **관리자 회원 CRUD** — 지점 관리자·전역 관리자 모두 가능(권한은 로그인 시 DB의 `admins.role`로 판정, 지점 관리자는 자기 지점 회원만). 필드: 이름(1~100자), 전화번호(숫자 11자리, 예 `01012345678`, 같은 지점 내 중복 금지·다른 지점 가입은 허용), 생년월일(필수), 소속 지점. **소속 지점은 변경 불가** — 회원이 다른 지점으로 옮기면 그 지점에 신규 등록한다. **삭제는 soft delete**(`deleted_at` 세팅). **관리자 화면에서는 풀 전화번호·생년월일을 마스킹 없이 표시한다(키오스크 `MemberPick`에서만 마스킹).**
4. **관리자 회원권 관리** — 부여, 정지, 정지 조기 활성화(unpause), 미래 예약 정지 취소(cancel-pause), 환불. 변경 기능은 없음.
   - **부여**: 월정액(`monthly` + `months` 입력, `end_date = start_date + months month`) / 횟수권(`pass10`, `start_date + 2 month` 자동·`remaining=10`). `start_date`는 오늘 또는 미래만 허용(과거 부여 불가). 결제 정보(금액 양수·수단[현금/카드]·결제일)를 함께 입력해 매출에 반영. 0원/무료 결제는 MVP 미지원.
   - **다음 회원권 미리 등록**: 만료 임박 시점에 다음 회원권을 미리 결제·등록할 수 있다. 단 **기간이 기존 active/paused 회원권과 겹치지 않아야** 한다(예: 5/30 만료 + 6/1~ 시작 OK, 5/29~ 시작은 거부 → 409 `MEMBERSHIP_PERIOD_OVERLAP`). 미래 시작 회원권은 시작일 도래 전까지는 체크인 사용 불가(시작 전 체크인 시 422 `MEMBERSHIP_NOT_STARTED`).
   - **정지**: 시작일·종료일·사유 입력 → 만료일이 정지 기간만큼 연장 + `pause_used=true`(한 회원권당 정지 1회 제한). 시작일이 오늘이면 즉시 `paused`로 전환, 미래면 `active` 유지(자정 배치가 도래일에 `paused`로 전환).
   - **조기 활성화(unpause)**: 정지가 도달해(`status='paused'`) 적용 중인 상태에서 회원이 일찍 활성화를 요청하면 관리자가 즉시 활성화. `actual_pause_end = 오늘`로 잡고 `end_date -= (pause_end_date - actual_pause_end)`로 만료일을 단축. 예: 4/1~4/7 정지·원 만료 5/30 → 4/6에 unpause 호출 시 만료 5/29.
   - **미래 예약 정지 취소(cancel-pause)**: 정지를 등록했지만 아직 도달하지 않은 상태(`status='active'` + `pause_used=true` + `pause_start_date > 오늘`)에서 호출. 등록 시 늘렸던 만큼 만료일을 되돌리고 `pause_used=false`로 초기화해 다시 정지 등록 가능 상태로 만든다.
   - **환불**: 사유 필수, `status='refunded'`로 전환. 환불 row의 `paid_at`은 환불 처리일(서버 KST 오늘)로 자동 기록되어 그날 매출에 음수 반영. 호출 가능 status: `active`, `paused`, `active`+미래 시작(사전 결제 후 마음 바꿈). **`expired`는 환불 불가**(409 `MEMBERSHIP_ALREADY_EXPIRED`). **MVP는 전체 환불만 지원**(부분 환불은 Phase 5+).
   - **체크인 가능 조건**: `status='active'` + `start_date <= 오늘` + `end_date >= 오늘`인 회원권이 있을 때만. `paused`(정지 기간 중)/`expired`/`refunded`/없음/시작 전(미래 등록)은 모두 체크인 차단. **영업시간 외 차단은 MVP에서 적용하지 않는다.**
   - **상태 자동 전환(자정 KST 배치)**: `active` AND `end_date` 경과 → `expired`. `paused` AND `pause_end_date` 경과 → `active`. `active` AND `pause_start_date` 도래 → `paused`. 만료일 당일은 그대로 사용 가능.
5. **대량 만료 연장** — 전역 관리자 전용. 지점·회원권 유형 필터 + 연장 일수 + 사유로 매칭된 활성 회원권 end_date를 일괄 연장(예: 연휴 3일 일괄 추가). 승인 절차·알림 없음.
6. **체크인 이력** — 오늘/기간별 조회. **`aggregate=raw|daily` 옵션** — `raw`(기본)는 모든 체크인 row, `daily`는 회원별 일자별 1회로 집계.
7. **매출 조회** — 전역 관리자 전용. 일일/월 매출(결제 건 합계, 수단별 분리) 조회. 지점 필터 가능. 풀 통계 대시보드는 MVP 제외.
8. **관리자 인증·계정 관리** —
   - 최초 로그인 시 비밀번호 변경 강제(`must_change_password=true`인 계정은 변경 화면 외 접근 차단).
   - 토큰 정책: access JWT 30분, refresh JWT 15시간. access 만료 시 refresh로 자동 갱신. 로그아웃·비번 변경·계정 soft delete·관리자 `branch_id` 변경 시 refresh 토큰 서버측 무효화.
   - 본인 비밀번호 변경은 **현재 비밀번호 재입력**으로 본인 인증(이메일·SMS·2FA 미사용, ADR-006). 강도: 8자 이상 + 영문/숫자 혼합. 변경 성공 시 해당 계정의 모든 refresh 토큰 무효화.
   - **로그인 방어**: 동일 계정 비번 5회 연속 실패 시 15분 잠금. 잠금 동안에는 정확한 비번도 거부. IP 단위 rate limit 추가.
   - **임시 비밀번호 만료**: 임시 비번은 **발급 후 24시간**까지 유효. 이후 로그인 시 401 `TEMP_PASSWORD_EXPIRED` — 전역 관리자가 다시 reset-password를 호출해 새 임시 비번을 발급해야 한다.
   - 비밀번호 분실 시 전역 관리자가 지점 관리자 계정에 임시 비밀번호(12자리 영숫자, 헷갈리는 문자 제외)를 발급하면 `must_change_password=true`가 자동 세팅되어 다음 로그인에서 강제 변경 흐름을 다시 탄다. 전역 관리자 본인 분실은 DB 직접 해시 갱신으로 처리(MVP 범위, OPERATIONS.md 참고).
   - **감사 로그**: 로그인 성공/실패, 로그아웃, 비번 변경, 비번 리셋, 관리자 CRUD, 지점 CRUD는 `admin_audit_logs`에 자동 기록(보안·운영 추적).
9. **지점 관리** — 전역 관리자가 지점 CRUD. 지점 이름 1~50자. **삭제는 soft delete**(`branches.deleted_at` 세팅). 해당 지점에 속한 활성(`deleted_at IS NULL`) 회원 또는 관리자가 1명이라도 존재하면 삭제를 차단한다(데이터 정합성 보호). 삭제된 지점은 모든 조회에서 `deleted_at IS NULL` 필터로 제외. 지점 주소(`branches.address`)는 unique(NULL 허용·빈 문자열 거부).
10. **관리자 계정 CRUD** — **전역 관리자 전용**. 지점 관리자 계정 생성(아이디·초기 비번·소속 지점·역할), 정보 수정(아이디·역할·소속 지점), 목록 조회, 삭제(soft delete), 비번 리셋. 생성·리셋 시 `must_change_password=true`로 시작해 첫 로그인에서 강제 변경. `branch_id` 변경 시 해당 사용자 refresh 토큰 무효화. 본인 계정의 role/branch_id 변경은 차단(`CANNOT_MODIFY_SELF_ROLE`). 전역 관리자 본인은 시드로만 생성(MVP 범위).

## MVP 제외 사항
- 회원이 개인 스마트폰으로 체크인
- 수업/프로그램·코치 지정
- 결제 게이트웨이 연동·구독 자동 청구(매출 기록은 수기 입력으로만 받는다)
- SMS·카카오 알림
- 지점 간 크로스 체크인
- 오프라인 큐잉(네트워크 단절 시 체크인)
- 통계 대시보드(전역 관리자용 시각화·기간 비교 등 — 매출 합계 조회만 MVP에 포함, 본격 대시보드는 차후)
- 네이티브 앱(관리자 UI는 반응형 웹앱으로 대체)

## 키오스크 운영(브라우저 UI 숨김)
회원에게 주소창·탭이 보이지 않게 하려면 다음을 적용한다.
- **MVP**: PWA로 배포 — `manifest.json`에 `"display": "fullscreen"` 지정 후 태블릿 홈 화면에 추가. 추가된 아이콘으로 진입하면 브라우저 크롬 없이 풀스크린으로 실행된다.
- **운영 강화 시**: Android는 Fully Kiosk Browser 같은 키오스크 전용 앱, iOS는 Guided Access 모드로 회원이 닫지 못하게 잠근다. (MVP 범위 밖)

## 디자인
- **키오스크(회원 화면)**: 큰 터치 타겟, 1초 만에 이해되는 버튼, 짧은 체류 시간. 고대비.
- **관리자(웹앱)**: 도구처럼. 데이터 테이블·폼 중심. 반응형으로 PC·모바일 모두 지원.
- **색상**: 무채색 배경 + 포인트 1색. 보라/인디고·그라디언트 텍스트·glass morphism 금지.
